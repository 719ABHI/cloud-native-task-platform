# Cloud Native Task Management Platform

## Project Overview

The **Cloud Native Task Management Platform** is a backend-focused project designed to demonstrate modern software engineering and cloud-native development practices. The project is being developed incrementally through well-defined sprints, starting with a Spring Boot REST API and progressively incorporating DevOps technologies such as Docker, Kubernetes, GitHub Actions, Terraform, AWS, and observability tools.

Beyond implementing application features, this project focuses on applying industry-standard software engineering practices, cloud-native technologies, and DevOps workflows commonly used in modern backend systems.

---

## Project Goals

This project aims to:

- Build a scalable RESTful backend using Spring Boot.
- Follow layered architecture and clean coding principles.
- Apply backend best practices such as DTOs, validation, centralized exception handling, logging, and API documentation.
- Containerize and orchestrate the application using Docker and Kubernetes.
- Automate build and deployment using GitHub Actions.
- Provision cloud infrastructure using Terraform and deploy the application to AWS.
- Implement monitoring, logging, and observability using Prometheus, Grafana, and Loki.
- Maintain structured documentation for each sprint, including implementation details, architectural decisions, and learning notes.

## Current Features

The following features have been implemented:

- RESTful CRUD APIs for task management
- Layered Spring Boot architecture (Controller → Service → Repository)
- PostgreSQL integration using Spring Data JPA and Hibernate
- DTO-based request and response handling
- Request validation using Bean Validation
- Global exception handling
- Custom business exceptions
- Unified API error responses
- Swagger/OpenAPI documentation
- Structured application logging using SLF4J

## Getting Started

### Prerequisites

- Java 21
- Gradle
- PostgreSQL

### Clone the Repository

```bash
git clone <repository-url>
cd cloud-native-task-platform
```

### Run the Backend

```bash
cd task-management-backend
./gradlew bootRun
```

The application will start on:

```
http://localhost:8080
```
